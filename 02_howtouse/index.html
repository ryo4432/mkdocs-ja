    使い方¶

    mkdocs serve¶

    127.0.0.1:8000に静的サイトを立ち上げてくれます。

    docker run --rm -it -p 8000:8000 "%cd%":/docs ryo4432/mkdocs:latest

    docker run --rm -it -p 8000:8000 ${PWD}:/docs ryo4432/mkdocs:latest

    docker run --rm -it -p 8000:8000 $(pwd):/docs ryo4432/mkdocs:latest

    mkdocs build¶

    siteフォルダ以下にhtmlファイルを生成してくれます。

    docker run --rm -it -p 8000:8000 "%cd%":/docs ryo4432/mkdocs:latest build

    docker run --rm -it -p 8000:8000 ${PWD}:/docs ryo4432/mkdocs:latest build

    docker run --rm -it -p 8000:8000 $(pwd):/docs ryo4432/mkdocs:latest build

    フォルダ構成¶

    フォルダ構成は以下の通り。マークダウンと構成ファイルであるmkdocs.ymlがあればOK。このおかげでmkdocs.ymlを後付けすれば既存のマークダウンに手を加えなくても静的サイトができる。docsフォルダに納めなくてもmkdocs.ymlでマークダウンの保存先フォルダを指定できるのでこの構成でなければならないというわけではない。

    ├── docs
    │  ├── index.md
    │  ├── chap01.md
    │  ├── chap02.md
    │  ├── chap03.md
    │  ├── chap04.md
    │  └── pics
    │     ├── pic01.png
    │     └── pic02.png
    ├── mkdocs.yml
